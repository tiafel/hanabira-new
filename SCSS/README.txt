В этой папке лежат файлы SCSS, которые предназначены для упрощения правки и ассембла большого количества стилей одновременно без необходимости править каждый по отдельности.

Данный SCSS, конечно, далеко не идеален: сайт не разрабатывался одновременно с ним, что автоматически ставит выбор между потерей части оформления стилей, переписыванием фронтэнда или оболваниванием SCSS. Я пошёл по последнему пути.

Инструкция по компиляции:

1) Ставим интерпретатор. Под винду советую Compass-app, под линукс придётся накачивать Ruby и все библиотеки ручками - инструкция есть в интернете.
2) Задаём интерпретатору рабочую папку. Кладём SCSS в папку инпута, забираем готовые CSS из папки отпута.
3) При выпуске в продакшн ОБЯЗАТЕЛЬНО прогоняем результат через csstidy. Это устранит дублирование и уберёт девелоперские комментарии из продакшн-версии. Параметры ниже.

csstidy [имя_файла] --compress_colors=true --remove_last_;=true --merge_selectors=2 --optimise_shorthands=1 [имя_итогового_файла]
